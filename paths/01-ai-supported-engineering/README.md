# Path 1: AI-supported engineering

The reading order for an engineering manager or senior engineer who wants to roll AI tooling out at a real firm. The path covers selection, the operating model around the tools, and the meta-work the tools quietly create.

## Audience

EMs running 5-15 person squads. Senior and Staff engineers asked to lead an AI-tools rollout at the squad or guild level. Directors briefing their orgs in plain English without buying any specific vendor's framing.

## Tiers

The entries live in the tier files; each file holds two annotated resources with the why-this-slot reasoning.

- **[30 minutes](./30-minutes.md)**: Anthropic Claude Code documentation + Simon Willison's running posts.
- **[3 hours](./3-hours.md)**: `awslabs/aidlc-workflows` end-to-end + Will Larson's *An Elegant Puzzle*.
- **[30 hours](./30-hours.md)**: Stand up `agentic-harness` on one squad + author the squad's rollout document.
- **[Notes](./notes.md)**: why the path looks the way it does, what was considered and rejected, open questions.

## What this path is not

A vendor-comparison shopping list. The substrate-shape framing in `AIDLC.md` §3 covers the comparison better than any matrix that goes stale every quarter.

A finished book. The 30-hour tier asks for a shipped artefact, not a finished read. Reading without shipping is the failure mode the path is written to avoid.
